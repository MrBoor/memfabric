## Analysis

### What works well

1. **Open-domain questions (84.5%)** — the model excels at integrating general
   knowledge with stored facts. MemFabric's verbose memory entries give the LLM
   enough context to reason well.

2. **Consistent performance** — no conversation below 72.4%. The approach is
   robust across different conversation styles and topics.

3. **No infrastructure** — compared to Mem0 (Qdrant vector DB, embedding model)
   or MemMachine (vector DB + reranker), MemFabric uses only local files.

### Where it struggles

1. **Multi-hop questions (68.2%)** — require synthesizing facts from multiple
   sessions. When facts are spread across different memory files, the model
   sometimes reads the wrong files or doesn't read enough files.

2. **Temporal questions (69.8%)** — relative dates ("last week", "the Friday
   before") are often not converted to absolute dates during ingest, leading
   to imprecise temporal reasoning.

3. **conv-42 has only 3 memory files** — the LLM consolidated too aggressively,
   creating huge files instead of focused ones. This hurt retrieval precision.

### Key observations

- **File count matters:** conv-42 (3 files, 106KB) scored 76.9% while conv-30
  (6 files, 49KB) scored 87.7%. More focused, smaller files improve retrieval.

- **Haiku's ingest quality is good but not great.** It misses some specific
  details (exact book titles, pet species, number of children). A stronger
  ingest model (Sonnet, GPT-4o) would likely improve single-hop scores.

- **Query model reads ~2.5 files per question on average.** For multi-hop
  questions that require 3-4 files, this is insufficient.

### Potential improvements

1. **Reorganize after ingest** — split oversized files, ensure 8-15 focused
   files per conversation.
2. **Stronger ingest model** — Sonnet or GPT-4o for better detail extraction.
3. **Better ingest prompt** — explicitly instruct to capture ALL specific
   details: names, numbers, dates, book titles, pet species.
4. **Query prompt tuning** — instruct to read more files (3-4) and say
   "I don't know" rather than guessing.
5. **Date normalization** — convert relative dates to absolute during ingest.
